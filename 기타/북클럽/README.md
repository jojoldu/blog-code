# 향로와 함께한 북클럽 1기 회고 웹페이지 제작서

## 1. 제작 목적

향로와 함께한 북클럽 1기를 기념하는 회고 웹페이지를 제작한다.

이 페이지는 단순한 후기나 모집 페이지가 아니라, 8명이 8주 동안 4권의 책을 함께 읽고, 독후감과 북토크를 나누며 만든 시간을 하나의 "픽셀 RPG 게임 클리어 기록"처럼 보여주는 기념 페이지다.

핵심은 책 자체의 정보보다, 책을 사이에 두고 생겨난 대화, 기억, 질문, 참여자들의 흔적을 시각적으로 남기는 것이다.

## 2. 핵심 콘셉트

- 콘셉트: 픽셀 RPG 게임의 클리어 화면, 저장 파일, 파티 회고록
- 레퍼런스: [Spring I/O 2026](https://2026.springio.net/)
- 주요 분위기: 독서모임, 추억, 회고, 모험, 퀘스트 클리어
- 핵심 키워드:
  - `BOOK CLUB SAVE FILE`
  - `QUEST CLEAR`
  - `Party Members`
  - `World Map`
  - `Quest Log`
  - `Replay Theater`
  - `Dialogue Log`
  - `Memory Items`
  - `Ending Credit`

## 3. 결과물 형태

- 정적 HTML 단일 페이지
- CSS와 JavaScript는 HTML 내부에 포함해도 된다.
- 별도 빌드 과정 없이 브라우저에서 바로 열 수 있어야 한다.
- PC와 모바일 모두 대응해야 한다.
- 상단 내비게이션으로 각 섹션에 이동할 수 있어야 한다.

### 7.5 Quest Log

4권의 책을 퀘스트 카드로 정리한다.

섹션 설명:

```text
책 소개는 줄이고, 그 밤에 우리가 가져간 감각만 남겼습니다. 회고 페이지의 주인공은 책 자체보다, 책을 사이에 두고 생겨난 대화니까요.
```

책별 카드 내용:

#### Q1. 「어떻게 사랑할 것인가」

- 회차: 첫 번째 밤
- 설명: 장영희 교수님의 강의록을 통해 문학의 힘, 공감의 힘, 나를 먼저 돌보는 일을 이야기했습니다.
- 키워드:
  - 나를 먼저 돌보는 법
  - 숫자로 측정할 수 없는 가치
  - AI 시대에도 계속 책을 읽는 이유

#### Q2. 「이와타 씨에게 묻다」

- 회차: 두 번째 밤
- 설명: 프로그래머 출신 경영자의 말에서, 계속하게 만드는 보상회로와 좋은 피드백의 조건을 꺼내보았습니다.
- 키워드:
  - 내가 보상을 느끼는 일
  - 제품을 사랑하는 마음
  - 솔직한 피드백을 가능하게 하는 태도

#### Q3. 「카이젠 저니」

- 회차: 세 번째 밤
- 설명: 애자일을 정답지가 아니라, 혼자 시작해 한 명의 동료에게 번지는 변화 방식으로 읽었습니다.
- 키워드:
  - 혼자 시작하는 작은 개선
  - 프랙티스보다 문제
  - 조직 안의 두꺼운 경계 넘기

#### Q4. 「달리기를 말할 때 내가 하고 싶은 이야기」

- 회차: 마지막 밤
- 설명: 북클럽이 끝난 뒤에도 각자의 리듬으로 계속 가기 위해, 자기 페이스와 힘을 남기는 법을 나눴습니다.
- 키워드:
  - 3미터 앞만 보기
  - 오래 가기 위해 멈추기
  - 나에게도 달리기 같은 것 찾기

### 7.6 Review Fragments

참여자들이 제출한 독후감을 전체 원문으로 싣기보다는, 북토크로 이어진 생각의 조각처럼 카드화한다.

섹션 설명:

```text
제출된 독후감은 원문 전체보다, 북토크로 이어진 생각의 조각으로 남겼습니다. 아래는 3회차 「카이젠 저니」 독후감을 익명화해 녹인 샘플입니다.
```

카드 예시:

1. `“이걸 기대하시는 게 맞나요?”`
   - 복명복창과 잦은 공유만으로는 충분하지 않았다. 혼자 추측한 과녁과 팀이 기대하는 과녁을 맞추기 위해, 1:1에서 기대를 직접 물어보는 실험이 시작됐다.

2. `문화는 직급이 아니라 움직임에서 시작된다.`
   - 워터폴 방식이 많은 조직 안에서도, “내가 할 수 있는 건 없다”는 마음을 내려놓고 AI 활용과 애자일 방식을 팀의 언어로 바꿔볼 수 있겠다는 확신이 생겼다.

3. `회사와 나의 목표가 다르지만은 않았다.`
   - 감사 카드, 플래닝 포커, 사용자 스토리 같은 활동을 다시 보니, 결국 팀이 같은 방향을 확인하고 경계를 넘기 위한 준비였다는 감각이 남았다.

4. `“어려운 점은 000입니다.”`
   - 데일리 스크럼에 어제 한 일과 오늘 할 일만 말하는 대신, 목표를 막는 어려움을 함께 말하기로 했다. 문제는 드러나야 더 커지지 않는다.

5. `한 사람의 열정보다, 함께 걷는 신뢰.`
   - “천릿길도 한 걸음부터”라는 문장은 한 사람의 끈기에서 끝나지 않았다. 공동의 목적을 신뢰하고, 모르는 것을 함께 묻는 사람들이 있을 때 작은 걸음은 팀의 여정이 된다.

### 7.8 Dialogue Log

발제문을 RPG 대화창처럼 표현한다.

요구사항:

- 왼쪽에는 질문 탭을 배치한다.
- 오른쪽에는 선택된 질문의 상세 내용이 대화창처럼 표시된다.
- 탭 클릭 시 질문, 설명, 선택지가 변경된다.

섹션 설명:

```text
발제문은 RPG의 대화창처럼 남겼습니다. 질문을 고르면, 그 질문이 열었던 대화의 방향을 다시 볼 수 있습니다.
```

질문 데이터:

#### 질문 1

- 스피커: `NPC 향로 · QUEST 1`
- 질문: 어떤 콘텐츠를 보고 감동을 받아서 실제 행동까지 바뀐 경험이 있나요?
- 설명: 첫 회차의 질문은 문학을 삶 밖의 교양이 아니라, 내 행동을 바꾸는 이야기로 다시 보게 했습니다. 감동과 변화 사이에는 지금 내 문제와 연결되는 순간이 있었습니다.
- 선택지:
  - A. 내 현재 문제와 연결된 이야기
  - B. 마음속에 쌓였다가 나중에 발현되는 이야기

#### 질문 2

- 스피커: `NPC 향로 · QUEST 2`
- 질문: 열심히 하는데 몸에 붙지 않는 일과, 힘든데도 계속하게 된 일은 무엇이 달랐나요?
- 설명: 이와타 사토루의 말에서 출발한 질문은 AI 시대의 성취감으로 이어졌습니다. 코드를 많이 쓰는 기쁨에서, 더 큰 문제를 찾고 임팩트를 만드는 기쁨으로 보상회로가 옮겨갈 수 있는지 물었습니다.
- 선택지:
  - A. 고생에서 오는 성취감
  - B. 임팩트에서 오는 성취감

#### 질문 3

- 스피커: `NPC 향로 · QUEST 3`
- 질문: 우리 팀에서 지금 당장 혼자 시작할 수 있는 작은 카이젠은 무엇인가요?
- 설명: 카이젠 저니의 핵심은 계몽보다 전염에 가까웠습니다. 가르치듯 바꾸려는 대신, 내가 필요해서 시작한 작은 습관이 한 명의 동료에게 옮겨붙는 순간을 떠올렸습니다.
- 선택지:
  - A. 혼자라도 시작할 개선
  - B. 함께할 첫 번째 동료

#### 질문 4

- 스피커: `NPC 향로 · QUEST 4`
- 질문: 지금은 3미터 앞만 보면 되는 때인가요?
- 설명: 마지막 회차의 질문은 북클럽 이후의 삶으로 이어졌습니다. 멀리 보면 움직이기 어려운 순간에는 오늘 할 수 있는 작은 행동이 필요하고, 동시에 그 3미터가 어느 방향인지 확인하는 일도 필요했습니다.
- 선택지:
  - A. 지금 당장 움직일 한 걸음
  - B. 방향을 잃지 않기 위한 점검

### 7.9 Memory Items

북클럽이 끝난 뒤에도 남는 것들을 RPG 인벤토리 아이템처럼 표현한다.

섹션 설명:

```text
북클럽이 끝나도 인벤토리에 남은 것들입니다. 각각의 아이템 위에 마우스를 올리면, 이 8주가 남긴 작은 효과를 볼 수 있습니다.
```

아이템 목록:

| 아이템 | 설명 |
| --- | --- |
| 밑줄 친 문장 | 책을 읽다가 멈춘 자리. 나중에 다시 보면 그때의 마음이 같이 저장되어 있습니다. |
| 600자 독후감 | 멋진 글보다 솔직한 글. 내가 무엇을 생각하는 사람인지 꺼내는 훈련이었습니다. |
| 발제 질문 | 정답을 찾기보다, 같이 오래 붙잡아볼 만한 질문을 얻었습니다. |
| 작은 카이젠 | 혼자 시작해도 괜찮지만, 언제까지나 혼자이면 안 된다는 감각. |
| 나만의 페이스 | 남들이 뭘 하든 상관없이, 오래 갈 수 있는 속도를 다시 묻는 힘. |
| 함께 읽은 밤 | 책은 혼자 읽었지만, 기억은 같이 만든다는 것을 알게 된 시간. |

### 7.10 Campfire

마지막 밤의 회고와 추억을 모닥불 장면으로 표현한다.

섹션 설명:

```text
마지막 밤의 모닥불입니다. 각자의 사적인 이야기는 덜어내고, 함께 남겨도 좋은 감정과 질문만 익명 로그처럼 모았습니다.
```

제목:

```text
서로에게 남은 흔적
```

회고 문장:

- “지금은 3미터 앞만 볼 때인지, 더 멀리 볼 때인지 구분해보고 싶어졌다.”
- “AI가 쉬운 문제를 더 쉽게 만들었다면, 나는 더 큰 문제를 찾아야 한다.”
- “혼자 시작한 작은 습관이 팀으로 번지는 순간이 있다는 걸 봤다.”
- “북클럽이 끝나도, 나에게 달리기 같은 것을 하나는 남겨두고 싶다.”

### 7.11 Ending Credit

게임 클리어 후 엔딩 크레딧처럼 마무리한다.

섹션 설명:

```text
클리어 화면은 끝이 아니라 저장 지점입니다. 다음 퀘스트는 각자의 자리에서 다시 시작됩니다.
```

제목:

```text
Thanks for Playing
```

본문:

```text
8주 동안 우리는 책을 완독하는 것보다, 책 앞에서 멈춘 자기 생각을 꺼내는 일을 더 오래 연습했습니다.

누군가는 AI 시대의 불안을 말했고, 누군가는 커리어의 방향을, 누군가는 건강과 루틴과 사람 사이의 거리를 말했습니다. 네 권의 책은 결국 각자의 삶을 다시 읽게 하는 맵이 되었습니다.

북클럽은 끝났지만, 각자의 달리기는 계속됩니다. 이 페이지가 언젠가 다시 필요한 순간에 조용히 열리는 저장 파일이 되기를 바랍니다.
```

## 8. 디자인 요구사항

### 8.1 전체 스타일

- 픽셀 RPG 게임 UI
- 게임 포스터, 티켓, 저장 파일, 월드맵, 퀘스트 카드, 대화창, 인벤토리, 엔딩 크레딧 같은 구조
- 굵은 검정 테두리
- 또렷한 픽셀 그림자
- 연한 격자 배경
- 각 섹션은 명확히 구분하되, 하나의 게임 세계처럼 이어져야 한다.

### 8.2 색상

주요 색상:

- 진한 남색/검정: 텍스트, 테두리, 대화창 배경
- 하늘색: 그림자, 강조
- 핑크: 클리어 문구, 강조 카드
- 노랑: 티켓, 버튼, 강조 라벨
- 초록: 북클럽/성장 느낌
- 보라: 보조 강조

권장 CSS 변수 예시:

```css
:root {
  --ink: #132536;
  --ink-deep: #07131e;
  --paper: #f7fbff;
  --paper-2: #e8f0f7;
  --blue: #53c4f3;
  --blue-dark: #1595cc;
  --pink: #ef4b6e;
  --pink-dark: #b62f4d;
  --yellow: #ffd85a;
  --yellow-dark: #dcae2c;
  --green: #60c653;
  --purple: #8f78dc;
}
```

### 8.3 타이포그래피

- 큰 영문 타이틀은 Impact 계열 또는 굵은 Condensed 계열 사용
- 한국어 본문은 `Pretendard`, `Apple SD Gothic Neo`, `Malgun Gothic`, system-ui 계열 사용
- 라벨, 번호, 작은 게임 UI 텍스트는 monospace 사용

폰트 예시:

```css
--body-font: "Pretendard", "Apple SD Gothic Neo", "Malgun Gothic", system-ui, sans-serif;
--display-font: Impact, Haettenschweiler, "Arial Narrow Bold", "Arial Black", sans-serif;
--mono-font: "Courier New", "D2Coding", monospace;
```

### 8.4 레이아웃

- 최대 콘텐츠 너비: 약 1160px
- 모바일에서는 모든 그리드가 1열로 자연스럽게 내려가야 한다.
- YouTube 영상은 16:9 비율 유지
- 텍스트가 버튼이나 카드 밖으로 넘치지 않아야 한다.

## 9. 인터랙션 요구사항

### 9.1 World Map

퀘스트 노드 클릭 시 다음 요소를 변경한다.

- 기간
- 장소
- 책 제목
- 설명 문구
- active 상태

### 9.2 Dialogue Log

질문 탭 클릭 시 다음 요소를 변경한다.

- 스피커
- 질문
- 설명
- 선택지
- active 상태
- `aria-selected` 값

### 9.3 Memory Items

- hover 또는 focus 시 설명 툴팁 표시
- 키보드 접근성을 위해 `tabindex="0"` 적용

## 10. 접근성 요구사항

- `lang="ko"` 설정
- `meta viewport` 설정
- 상단에 본문 바로가기 링크 제공
- 각 섹션은 `aria-labelledby` 사용
- 인터랙션 영역에는 적절한 `aria-label`, `aria-live`, `role="tablist"`, `role="tab"` 사용
- 이미지에는 alt 제공
- 버튼과 링크는 키보드로 접근 가능해야 한다.
