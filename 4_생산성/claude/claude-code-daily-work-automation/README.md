# Claude Code로 일상 업무 반자동화하기

> 이 글은 [Claude Codeですべての日常業務を爆速化しよう！](https://qiita.com/minorun365/items/114f53def8cb0db60f47)의 접근법을 기반으로, 직접 따라할 수 있는 실습 위주로 구성했다.

Claude Code를 아직 "코드 작성 도구"로만 쓰고 있다면, 이 글에서 소개하는 3가지를 직접 세팅해보자.

- "**업무 자동화 모노레포**" 만들기
- "**반복 업무용 Skill**" 만들기
- "**CSV 분석 → 보고서 자동 생성**" Skill 만들기

전부 터미널에서 Claude Code만으로 진행한다.

## 1. 업무 자동화 모노레포 만들기

코딩 프로젝트가 아닌 "**일상 업무**"를 Claude Code와 함께 처리하려면, 먼저 작업 공간이 필요하다.
핵심 아이디어는 간단하다. "**잡무를 전부 하나의 Git 리포지토리에 모아두는 것**"이다.

Claude Code는 리포지토리 단위로 컨텍스트를 갖는다.
즉, 모노레포를 열면 과거에 처리했던 모든 업무의 절차서·노하우·스크립트에 접근할 수 있다.
"지난달에 어떻게 했더라?"를 Claude Code에 물어보기만 하면 된다.

### Step 1: 디렉토리 생성

```bash
mkdir -p ~/work-automation
cd ~/work-automation
git init
```

### Step 2: 기본 구조 만들기

```bash
mkdir -p tasks/매월-경비정산
mkdir -p tasks/매월-근무보고
mkdir -p snippets
mkdir -p docs
```

완성되면 이런 구조가 된다:

```
work-automation/
├── CLAUDE.md              # Claude Code에 전달할 프로젝트 지시서
├── .claude/
│   └── skills/            # 반복 업무용 Skill 모음
├── tasks/
│   ├── 매월-경비정산/       # 반복 작업
│   │   ├── README.md       # 영구 절차서
│   │   └── 2026-03.md      # 이번 달 메모
│   └── 매월-근무보고/       # 반복 작업
│       └── README.md
├── snippets/               # 재사용 스크립트
└── docs/                   # 조사 메모
```

### Step 3: CLAUDE.md 작성

이 리포지토리에서 Claude Code가 알아야 할 "**사람만 아는 정보**"를 적는다.
코드에서 읽어낼 수 있는 건 적지 않는다.

```markdown
# 업무 자동화 리포지토리

## 이 리포지토리의 목적
일상 업무(경비 정산, 근무 보고, 자료 작성 등)를 Claude Code와 함께 반자동화하기 위한 작업 공간이다.

## 디렉토리 규칙
- tasks/매월-*: 매달 반복하는 업무. README.md에 영구 절차서, YYYY-MM.md에 해당월 메모
- tasks/YYYY-MM-*: 단발성 작업
- snippets/: 재사용 가능한 스크립트
- docs/: 조사 및 참고 메모

## 작업 원칙
- 100% 자동화를 목표로 하지 않는다. "가장 힘든 부분만 자동화 + 나머지는 수동"이면 충분하다.
- 새 작업을 처리했으면 반드시 README.md를 남긴다.
- 삽질했으면 README의 "주의점" 섹션에 즉시 추가한다.
- 같은 작업을 다음 달에도 한다면 tasks/매월-* 디렉토리로 승격한다.

## 출력 형식
- 외부 시스템에 입력할 데이터는 Markdown 테이블로 출력한다 (복사-붙여넣기 용이).
- 금액은 반드시 원(₩) 또는 엔(¥) 등 통화 단위를 명시한다.
```

이 파일을 `~/work-automation/CLAUDE.md`에 저장한다.

### Step 4: 경비 정산 절차서 예시

`tasks/매월-경비정산/README.md`:

```markdown
# 매월 경비 정산

## 절차
1. 카드 이용 내역 CSV를 다운로드하여 이 디렉토리에 저장
2. Claude Code에 "이번 달 경비 정산 해줘" 요청
3. 출력된 Markdown 테이블을 회계 시스템에 복사-붙여넣기
4. 신청 완료 후 해당월 .md에 메모

## 카드 이용 내역 CSV 형식
- 파일명: YYYY-MM-카드내역.csv
- 필수 컬럼: 이용일, 이용처, 금액, 카드종류

## 계정 과목 매핑
- SaaS/클라우드 서비스 → 통신비
- 도서 구매 → 도서인쇄비
- 교통비 → 여비교통비
- 식대 (업무) → 복리후생비
- 장비/기기 → 소모품비

## 주의점
- 개인 사용 건은 제외 표시할 것
- 10만원 이상 건은 영수증 첨부 필요 여부를 표시할 것
```

여기서 핵심은 "**CLAUDE.md와 각 디렉토리의 README.md가 곧 업무 인수인계 문서**"가 된다는 점이다.
다음 달에 Claude Code를 열면, 이 문서들을 참조해서 바로 작업에 들어간다.

## 2. 반복 업무용 Skill 만들기

Skill은 Claude Code의 "**재사용 가능한 작업 정의**"다.
`/` 슬래시 커맨드로 호출할 수 있어서, 매번 같은 프롬프트를 입력할 필요가 없다.

### 경비 정산 Skill 만들기

```bash
mkdir -p ~/work-automation/.claude/skills/expense-report
```

`.claude/skills/expense-report/SKILL.md`:

```markdown
---
description: 매월 경비 정산 처리
applyTo: "tasks/매월-경비정산/**"
---

# 경비 정산 처리

## 절차

1. `tasks/매월-경비정산/` 디렉토리에서 가장 최근 CSV 파일을 찾는다
2. CSV를 파싱하여 각 거래를 분류한다
3. `tasks/매월-경비정산/README.md`의 "계정 과목 매핑" 규칙에 따라 과목을 자동 매핑한다
4. 아래 형식의 Markdown 테이블로 출력한다
5. 개인 사용으로 의심되는 건이 있으면 별도 표시한다
6. 10만원 이상 건은 "영수증 확인 필요" 표시를 추가한다

## 출력 형식


| No | 이용일 | 이용처 | 금액 | 계정과목 | 적요 | 비고 |
|----|--------|--------|------|----------|------|------|
| 1  | 03/01  | AWS    | ₩50,000 | 통신비 | 클라우드 이용료 | |
| 2  | 03/05  | 교보문고 | ₩25,000 | 도서인쇄비 | 기술 서적 | |
```

## 최종 확인 요청

테이블 출력 후, 반드시 사용자에게 다음을 확인받는다:
- 분류가 맞는지
- 제외할 건이 있는지
- 적요를 수정할 건이 있는지

이제 Claude Code에서 이 리포지토리를 열고 `/expense-report`만 입력하면, CSV를 찾아서 분석하고 테이블까지 만들어준다.

### 근무 보고 Skill 만들기

```bash
mkdir -p ~/work-automation/.claude/skills/work-report
```

`.claude/skills/work-report/SKILL.md`:

```markdown
---
description: 매월 근무 보고서 작성
applyTo: "tasks/매월-근무보고/**"
---

# 근무 보고서 작성

## 절차

1. 사용자에게 이번 달 주요 업무를 질문한다: "이번 달 주요 업무를 알려주세요 (음성입력이나 간단한 메모도 OK)"
2. 입력받은 내용을 아래 형식으로 구조화한다
3. `tasks/매월-근무보고/README.md`에 정의된 보고서 양식이 있으면 그에 맞춘다

## 출력 형식

# YYYY년 MM월 근무 보고서

## 주요 업무
| 업무 | 내용 | 비중(%) |
|------|------|---------|

## 성과 요약
-

## 다음 달 계획
-

## 주의사항
- 사용자가 입력한 내용 외에 임의로 업무를 추가하지 않는다
- "잘한 점"이나 "개선할 점" 같은 평가성 문구를 마음대로 넣지 않는다
- 사용자의 메모가 구어체/음성입력이라도, 취지를 살려서 비즈니스 문체로 정리한다

```

## 3. CSV 분석 → 보고서 자동 생성 Skill 만들기

경비 정산 외에도, "**CSV 데이터를 받아서 분석 보고서를 만드는 작업**"은 어디서든 쓸 수 있다.
범용 CSV 분석 Skill을 하나 만들어두면 다양한 업무에 재활용할 수 있다.

```bash
mkdir -p ~/work-automation/.claude/skills/analyze-csv
```

`.claude/skills/analyze-csv/SKILL.md`:

```markdown
---
description: CSV 파일을 분석하여 요약 보고서를 생성한다
---

# CSV 분석 보고서 생성

## 절차

1. 사용자가 지정한 CSV 파일을 읽는다
2. 컬럼 구조와 데이터 타입을 파악한다
3. 기본 통계를 산출한다:
   - 행 수, 컬럼 수
   - 숫자 컬럼: 합계, 평균, 최대, 최소
   - 문자 컬럼: 고유값 수, 최빈값
   - 날짜 컬럼: 기간 범위
4. 이상값이나 빈 값이 있으면 보고한다
5. 사용자에게 "어떤 관점에서 분석할까요?"를 질문한다
6. 답변에 맞춰 추가 분석을 진행한다

## 분석 시 사용 가능한 도구
- Python (pandas, matplotlib): CSV 파싱과 집계에 사용
- 차트가 필요하면 matplotlib로 PNG 생성 후 파일로 저장

## 출력 형식
- Markdown 보고서로 출력한다
- 핵심 수치는 테이블로 정리한다
- 차트는 별도 파일로 저장하고 경로를 표시한다

## 주의사항
- 원본 CSV 파일은 절대 수정하지 않는다
- 데이터에 개인정보(이름, 전화번호 등)가 포함되어 있으면 경고한다
```

### 실제 사용 예시

터미널을 열고 `~/work-automation` 디렉토리에서 Claude Code를 실행한다:

```bash
cd ~/work-automation
claude
```

그리고 이렇게 말하면 된다:

```
이번 달 카드 내역 CSV를 tasks/매월-경비정산/ 에 넣어뒀어. 경비 정산 해줘.
```

또는 슬래시 커맨드로:

```
/expense-report
```

Claude Code가 CSV를 찾아서 파싱하고, README.md의 매핑 규칙에 따라 분류하고, Markdown 테이블을 출력한다.
그걸 회계 시스템에 복사-붙여넣기하면 끝이다.

## 핵심 원칙: 100% 자동화가 아니라 반자동화

이 모든 세팅의 밑바탕에 있는 원칙이 하나 있다.

"**100% 자동화를 목표로 하지 않는다.**"

현실에서는 권한 부족, 보안 정책, 레거시 시스템 등의 제약으로 완전 자동화가 불가능한 경우가 훨씬 많다.
회계 시스템에 API 쓰기 권한이 없을 수도 있고, 사내 시스템이 SaaS 연동을 금지하고 있을 수도 있다.

그래도 괜찮다.
"**가장 귀찮은 부분만 자동화하고, 마지막 단계는 사람이 복사-붙여넣기를 한다.**"
이 "반자동화" 전략이 실용적이면서도 가장 큰 효과를 낸다.

30분 걸리던 경비 정산이 5분으로 줄었다면, 나머지 25분의 가치는 충분하다.
그리고 그 절차가 리포지토리에 문서로 남으니, 다음 달엔 더 빨라진다.

## 다음 단계: 여기서 더 확장하려면

위 3가지를 세팅한 뒤, 자연스럽게 확장할 수 있는 방향들이다:

- "**Marp Skill 추가**": 회의록이나 메모를 Claude Code에 넘기면 Marp 슬라이드(.md)로 변환 → `marp --pdf`로 PDF 출력
- "**MCP 서버 연동**": Google Calendar MCP로 근무 일수 자동 카운트, Todoist MCP로 태스크 관리
- "**터미널 병렬 작업**": 터미널 창을 여러 개 열고 각각 다른 Skill을 돌리면, 승인 대기 중에 다른 작업 진행 가능

모두 같은 모노레포 안에서 Skill과 문서를 추가하는 것만으로 확장된다.
