# 맥북 숏컷 활용기: 메신저 문장에서 URL만 뽑아 붙여넣기

메신저나 문서에서 무언가를 복사하다 보면, 내가 필요한 값보다 더 많은 문구가 함께 클립보드에 들어올 때가 있다.

예를 들면 이런 식이다.

```text
리뷰 부탁드립니다.

PR: https://github.com/example/app/pull/123

오늘 배포 전에 확인되면 좋겠습니다.
```

내 개인 작업 노트나 Todo 앱에는 URL 하나만 기록해두고 싶은데, 매번 붙여넣은 뒤 앞뒤 문장을 지우는 일이 반복된다.

```text
https://github.com/example/app/pull/123
```

한두 번이면 괜찮다.
그런데 메모, 문서, 개인 노트에 자주 정리하다 보면 이 작은 편집이 계속 거슬린다.

이번에 만든 것은 다른 사람에게 보낼 문구를 바꾸는 도구가 아니라, 내 작업 노트에 필요한 값만 빠르게 정리하기 위한 클립보드 자동화다.
macOS의 **단축어(Shortcuts)** 앱을 이용해서, 클립보드에 들어 있는 텍스트에서 URL만 찾아 다시 클립보드에 넣고 바로 붙여넣는 흐름을 만들었다.

처음에는 생각보다 간단할 줄 알았다.
실제로 URL을 추출해서 클립보드에 다시 넣는 것까지는 금방 됐다.
문제는 마지막 자동 붙여넣기였다.

단축키로 숏컷을 실행하면 URL은 클립보드에 잘 들어가는데, 붙여넣기가 되지 않았다.
권한 문제도 있었고, 단축키를 누른 손이 떼어지기 전에 `Cmd + V`가 다시 들어가면서 붙여넣기가 씹히는 문제도 있었다.

그래서 이 글은 완성된 설정만 적기보다, 실제로 막혔던 지점과 수정 과정을 같이 남겨둔다.

## 최종 동작

완성 후 사용 흐름은 다음과 같다.

```text
1. 메신저나 문서에서 텍스트를 복사한다.
2. 붙여넣고 싶은 곳으로 이동한다.
3. 지정한 단축키를 누른다.
4. 숏컷이 클립보드에서 첫 번째 URL만 추출한다.
5. 추출한 URL을 클립보드에 다시 복사한다.
6. 0.2초 뒤 자동으로 붙여넣는다.
```

예를 들어 클립보드에 아래 내용이 들어 있어도,

```text
리뷰 부탁드립니다.

PR: https://github.com/example/app/pull/123

오늘 배포 전에 확인되면 좋겠습니다.
```

숏컷을 실행하면 아래 URL만 붙여넣어진다.

```text
https://github.com/example/app/pull/123
```

## 왜 브라우저 확장이 아니라 macOS 숏컷인가?

이 문제는 크롬 확장 프로그램으로도 해결할 수 있다.
특정 페이지의 버튼 클릭을 감지하고, 복사된 클립보드를 다시 정리하는 방식이다.

하지만 처음부터 확장 프로그램을 만들면 설치, 권한, 브라우저별 관리까지 생각해야 한다.

반면 macOS 숏컷은 가볍다.

- 별도의 개발 환경이 필요 없다.
- macOS 기본 앱만으로 만들 수 있다.
- 크롬뿐 아니라 Safari, 메모장, Slack, Notion 등 어느 앱에서든 사용할 수 있다.
- 단축키 하나로 "클립보드 정리 + 붙여넣기"까지 처리할 수 있다.

이 문제는 브라우저 안에서 해결하기보다 **붙여넣기 직전에 클립보드를 정리하는 방식**으로 풀면 충분하다.

## 만들 숏컷의 핵심 로직

숏컷이 할 일은 단순하다.

```text
클립보드 가져오기
→ 정규식으로 URL 찾기
→ 첫 번째 URL 가져오기
→ 찾은 URL만 클립보드에 다시 복사
→ 0.2초 기다리기
→ Cmd + V 실행
```

이번 예시에서는 클립보드에 들어 있는 텍스트에서 첫 번째 URL만 추출한다.

사용할 정규식은 다음과 같다.

```regex
https?://[^\s]+
```

이 정규식은 아래와 같은 URL을 찾아낸다.

```text
https://github.com/example/app/pull/123
https://www.notion.so/example-meeting-note
http://localhost:3000/debug
```

더 엄격하게 특정 서비스의 URL만 찾고 싶다면 이 정규식만 바꾸면 된다.

## 1. 단축어 앱 열기

먼저 macOS에서 **단축어(Shortcuts)** 앱을 연다.

Spotlight에서 아래처럼 검색하면 바로 열 수 있다.

```text
단축어
```

영어 환경이라면 다음 이름으로 보일 수 있다.

```text
Shortcuts
```

앱을 열었다면 오른쪽 위의 `+` 버튼을 눌러 새 단축어를 만든다.

단축어 이름은 예를 들어 다음처럼 정했다.

```text
URL만 붙여넣기
```

## 2. 빠른 동작 입력 설정하기

단축어 맨 위에는 빠른 동작 입력 블록이 있다.
내 경우에는 아래처럼 두었다.

```text
빠른 동작에서 앱 및 18개 받기
입력이 없는 경우: 계속하기
```

이 숏컷은 선택한 텍스트를 받아 처리하는 용도라기보다, 이미 클립보드에 들어 있는 텍스트를 처리하는 용도다.
그래서 입력이 없어도 계속 진행하도록 둔다.

## 3. 클립보드 가져오기

첫 번째 실제 액션으로 클립보드 내용을 가져온다.

검색창에서 다음 액션을 찾는다.

```text
클립보드 가져오기
```

영어 환경이라면 다음 액션이다.

```text
Get Clipboard
```

이 액션은 현재 클립보드에 들어 있는 텍스트를 다음 액션으로 전달한다.

## 4. 텍스트 일치 액션으로 URL 찾기

다음으로 정규식을 이용해 클립보드 내용에서 URL만 찾는다.

검색창에서 다음 액션을 추가한다.

```text
텍스트 일치
```

영어 환경에서는 다음 이름이다.

```text
Match Text
```

그리고 패턴 입력칸에 아래 정규식을 넣는다.

```regex
https?://[^\s]+
```

이 액션은 클립보드 텍스트 안에서 `http://` 또는 `https://`로 시작하는 URL을 찾아준다.

## 5. 첫 번째 결과만 가져오기

`텍스트 일치` 액션의 결과는 리스트 형태다.
보통 이런 클립보드 정리에서는 필요한 URL이 하나만 들어 있으므로 첫 번째 결과만 가져오면 된다.

검색창에서 다음 액션을 추가한다.

```text
목록에서 항목 가져오기
```

영어 환경에서는 다음 이름이다.

```text
Get Item from List
```

설정은 다음처럼 맞춘다.

```text
첫 번째 항목 가져오기
```

영어 환경에서는 다음처럼 보일 수 있다.

```text
Get First Item
```

## 6. 추출한 URL을 클립보드에 복사하기

이제 정규식으로 찾은 첫 번째 URL을 다시 클립보드에 넣는다.

검색창에서 다음 액션을 추가한다.

```text
클립보드에 복사
```

영어 환경에서는 다음 이름이다.

```text
Copy to Clipboard
```

이 단계까지 완료하면 클립보드에는 긴 문장이 아니라 아래 URL만 남게 된다.

```text
https://github.com/example/app/pull/123
```

여기까지만 해도 충분히 쓸 수 있다.
이 상태에서는 숏컷을 실행한 뒤 직접 `Cmd + V`를 누르면 된다.

처음에는 이 방식이 가장 안정적이다.
자동 붙여넣기까지 붙이면 편하긴 하지만, macOS 권한과 단축키 타이밍을 신경 써야 한다.

## 7. 자동 붙여넣기 추가하기

단축키 한 번으로 붙여넣기까지 끝내고 싶다면 마지막에 `AppleScript 실행` 액션을 추가한다.

검색창에서 다음 액션을 추가한다.

```text
AppleScript 실행
```

영어 환경에서는 다음 이름이다.

```text
Run AppleScript
```

처음에는 아래처럼 작성했다.

```applescript
on run {input, parameters}
  tell application "System Events"
    keystroke "v" using command down
  end tell
  return input
end run
```

그런데 이 방식은 내 환경에서 제대로 동작하지 않았다.
URL은 클립보드에 들어갔지만, 실제 붙여넣기가 되지 않았다.

## 8. 첫 번째 문제: 키스트로크 권한

처음 만난 오류는 아래 메시지였다.

```text
System Events에 오류 발생: 단축어에서 키스트로크를 보내도록 허용되지 않습니다.
```

이건 숏컷 로직 문제가 아니라 macOS 권한 문제다.
`System Events`로 `Cmd + V` 같은 키 입력을 보내려면 단축어 앱에 손쉬운 사용 권한이 필요하다.

권한은 아래 경로에서 켠다.

```text
시스템 설정
→ 개인정보 보호 및 보안
→ 손쉬운 사용
→ 단축어 허용
```

목록에 `단축어`가 없다면 `+` 버튼으로 직접 추가하면 된다.

```text
+ 버튼
→ 응용 프로그램
→ 단축어.app 선택
→ 추가
→ ON
```

처음에는 `개인정보 보호 및 보안 → 자동화` 쪽을 찾아봤는데, 내 환경에서는 그 목록에 `단축어`가 나오지 않았다.
이 경우에도 이상한 것은 아니었다.

자동화 목록은 앱이 다른 앱을 제어하는 권한 요청을 만든 뒤에 보이는 경우가 많다.
이번 문제의 핵심은 키보드 입력을 보내는 것이므로, 먼저 봐야 할 곳은 `자동화`가 아니라 `손쉬운 사용`이었다.

권한을 켠 뒤에는 단축어 앱을 완전히 종료했다가 다시 열었다.

## 9. 두 번째 문제: 단축키를 누른 손이 아직 안 떨어졌다

권한을 켜도 바로 끝나지는 않았다.
URL은 클립보드에 복사되는데 붙여넣기는 여전히 되지 않았다.

원인은 단축키 조합이었다.
나는 이 숏컷을 아래 단축키로 등록했다.

```text
Control + Option + Command + V
```

이 단축키로 숏컷을 실행한 직후 AppleScript가 바로 `Cmd + V`를 보내면, 손가락이 아직 `Control`, `Option`, `Command`에서 떨어지기 전일 수 있다.

그러면 의도한 입력은 `Cmd + V`인데, 실제로는 아래에 가까운 입력이 들어간다.

```text
Control + Option + Command + V
```

이러면 붙여넣기로 처리되지 않을 수 있다.

그래서 AppleScript 앞에 아주 짧은 지연 시간을 넣었다.
그리고 `keystroke "v"` 대신 물리 키 코드로 `V` 키를 보내도록 바꿨다.

최종적으로 잘 동작한 코드는 아래다.

```applescript
on run {input, parameters}
  delay 0.2
  tell application "System Events"
    key code 9 using {command down}
  end tell
  return input
end run
```

`key code 9`는 `V` 키다.
`using {command down}`은 `Command` 키를 누른 상태로 실행한다는 뜻이다.

`delay 0.2`는 0.2초, 즉 200ms를 기다린다는 뜻이다.
이 정도면 단축키를 누른 손이 떨어질 시간을 주면서도, 사용감이 크게 느려지지 않았다.

처음에는 `delay 0.8`로 테스트했다.
잘 동작했지만 체감상 조금 길었다.
이후 `0.4`, `0.3`, `0.25`, `0.2`로 줄여봤고, 내 환경에서는 `0.2`가 충분히 안정적이었다.

혹시 가끔 붙여넣기가 씹힌다면 `0.25`, `0.3`이나 `0.4`로 올리면 된다.

```applescript
delay 0.25
```

```applescript
delay 0.3
```

```applescript
delay 0.4
```

반대로 `0.2`보다 더 낮추면 다시 같은 문제가 생길 수 있다.

## 10. 최종 액션 구성

최종 숏컷 구성은 다음 순서가 된다.

```text
1. 클립보드 가져오기
2. 텍스트 일치
   패턴: https?://[^\s]+
3. 목록에서 항목 가져오기
   첫 번째 항목
4. 클립보드에 복사
5. AppleScript 실행
   on run {input, parameters}
     delay 0.2
     tell application "System Events"
       key code 9 using {command down}
     end tell
     return input
   end run
```

단축키는 아래처럼 지정했다.

```text
Control + Option + Command + V
```

오른쪽 세부사항에서는 빠른 동작으로 사용하도록 설정했다.

```text
빠른 동작으로 사용
서비스 메뉴
```

단축키를 다른 조합으로 지정해도 된다.
다만 기본 붙여넣기인 `Cmd + V`를 그대로 대체하려고 하기보다는, 별도의 "정리해서 붙여넣기" 단축키를 하나 두는 편이 안전하다.

## 11. 사용해보기

메신저에서 필요한 URL이 포함된 텍스트를 복사한다.

```text
리뷰 부탁드립니다.

PR: https://github.com/example/app/pull/123

오늘 배포 전에 확인되면 좋겠습니다.
```

붙여넣고 싶은 앱으로 이동해서 커서를 둔다.
그리고 `Control + Option + Command + V`를 누른다.

그러면 아래 값만 붙여넣어진다.

```text
https://github.com/example/app/pull/123
```

## 예외 상황 처리하기

위 방식은 클립보드에 URL이 하나 이상 있다고 가정한다.

만약 클립보드에 URL이 없는 상태에서 실행하면, `목록에서 항목 가져오기` 단계에서 결과가 없어서 숏컷이 멈출 수 있다.

더 안정적으로 만들고 싶다면 `텍스트 일치` 다음에 조건문을 추가할 수 있다.

```text
텍스트 일치 결과가 있는 경우
→ 첫 번째 항목 가져오기
→ 클립보드에 복사
→ AppleScript로 Cmd + V 실행

그렇지 않은 경우
→ 그냥 Cmd + V 실행
```

이렇게 구성하면 클립보드에 URL이 있을 때는 그 값만 정제해서 붙여넣고, 없을 때는 일반 붙여넣기처럼 동작한다.

다만 처음 만들 때는 조건문 없이 단순하게 시작하는 편이 낫다.
URL 추출, 클립보드 복사, 자동 붙여넣기가 각각 잘 되는지 따로 확인하기 쉽기 때문이다.

## 정규식 확장하기

현재 정규식은 가장 단순한 URL 패턴을 대상으로 한다.

```regex
https?://[^\s]+
```

이 패턴은 대부분의 URL을 쉽게 잡아내지만, URL 뒤에 마침표나 닫는 괄호가 붙은 문장에서는 그 문자까지 함께 잡을 수 있다.
그런 경우에는 정규식을 조금 더 좁히면 된다.

예를 들어 GitHub PR 링크만 찾고 싶다면 아래처럼 바꿀 수 있다.

```regex
https://github\.com/[^/\s]+/[^/\s]+/pull/[0-9]+
```

Notion 문서 URL만 찾고 싶다면 이런 식으로 바꿀 수도 있다.

```regex
https://www\.notion\.so/[^\s]+
```

다만 자동화는 처음부터 범위를 넓히는 것보다, 내가 반복해서 겪는 작은 불편 하나만 정확히 줄이는 편이 낫다.
처음에는 일반 URL 패턴으로 시작하고, 나중에 필요할 때 특정 서비스에 맞춰 좁히면 된다.

## 마무리

작은 자동화지만 자주 반복되는 작업일수록 효과가 크다.

복사한 텍스트에서 필요한 URL만 남기기 위해 앞뒤 문장을 지우는 일은 사소해 보인다.
하지만 개인 노트나 업무 메모에 같은 작업을 반복하다 보면 생각보다 자주 흐름이 끊긴다.

이번 숏컷에서 제일 중요했던 부분은 URL을 찾는 정규식보다, 마지막 붙여넣기를 안정적으로 실행하는 부분이었다.

```applescript
delay 0.2
key code 9 using {command down}
```

이 두 줄을 넣고 나서야 단축키 한 번으로 원하는 흐름이 완성됐다.

```text
복사
→ 정제해서 붙여넣기
```

맥북의 단축어 앱은 거창한 자동화보다 이런 작은 반복을 줄일 때 더 자주 쓰게 된다.
복사한 텍스트를 그대로 붙여넣는 대신, 붙여넣기 직전에 한 번 가공하는 것.
이 정도만 해도 손이 꽤 편해진다.
